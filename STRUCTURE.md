# 프로젝트 구조 설명

## 📁 폴더 구조

```
real_eunji/
├── src/                    # 개발용 소스 파일 (원본)
│   ├── css/
│   │   └── style.css      # 원본 CSS 파일
│   └── js/
│       └── script.js      # 원본 JavaScript 파일
│
├── dist/                   # 프로덕션용 배포 파일 (압축/난독화)
│   ├── css/
│   │   └── style.min.css  # 압축된 CSS 파일
│   └── js/
│       └── script.min.js  # 압축 및 난독화된 JavaScript 파일
│
├── assets/                 # 기존 자산 파일들 (호환성 유지)
│   ├── css/
│   │   ├── style.css      # 원본 CSS (src/css/와 동일)
│   │   └── style.min.css  # 압축 CSS (dist/css/와 동일)
│   ├── fonts/             # 폰트 파일들
│   ├── icons/             # 아이콘 파일들
│   ├── images/            # 이미지 파일들
│   └── js/
│       ├── script.js      # 원본 JS (src/js/와 동일)
│       └── script.min.js  # 압축/난독화 JS (dist/js/와 동일)
│
├── docs/                   # 문서 파일들
│   └── README.md
│
├── index.html             # 메인 HTML 파일 (dist/ 파일들을 참조)
├── README.md              # 프로젝트 설명
├── STRUCTURE.md           # 이 파일 - 구조 설명
└── vercel.json            # Vercel 배포 설정
```

## 🛠️ 개발/프로덕션 분리

### 개발용 파일 (`src/`)

- **원본 소스 코드**: 읽기 쉽고 주석이 포함된 원본 파일들
- **디버깅 용이**: 개발 시 수정하고 디버깅하기 편함
- **가독성 우선**: 코드 구조와 로직을 이해하기 쉽게 작성

### 프로덕션용 파일 (`dist/`)

- **압축 최적화**: 파일 크기를 최소화하여 로딩 속도 향상
- **난독화 처리**: JavaScript 코드를 읽기 어렵게 변환
- **성능 우선**: 웹사이트 성능과 보안을 고려한 최적화

## 🔒 보안 및 난독화

### CSS 압축

- 공백, 주석, 불필요한 문자 제거
- 선택자와 속성명 최소화
- 파일 크기 약 70% 감소

### JavaScript 난독화

- 변수명과 함수명을 `_0x1a2b` 형태로 변환
- 코드 구조를 해석하기 어렵게 변환
- 개발자도구에서 코드 분석 방지

## 📋 사용 방법

1. **개발 시**: `src/` 폴더의 파일들을 수정
2. **배포 시**: `dist/` 폴더의 압축된 파일들이 자동으로 사용됨
3. **HTML 파일**: `dist/` 경로의 파일들을 참조하도록 설정됨

## ⚡ 성능 개선

- **로딩 속도**: 압축된 파일로 인한 빠른 로딩
- **대역폭 절약**: 작은 파일 크기로 트래픽 절약
- **캐싱 효율**: 최적화된 파일들의 효율적인 브라우저 캐싱

## 🔄 업데이트 방법

1. `src/` 폴더의 원본 파일 수정
2. 압축/난독화 도구를 사용하여 `dist/` 폴더 업데이트
3. 변경사항이 자동으로 웹사이트에 반영됨
